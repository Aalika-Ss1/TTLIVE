# Competition Rules Draft

สถานะ: Draft สำหรับตรวจทานก่อนใช้จริง

เอกสารนี้เป็นดราฟกติกาที่เหมาะสมสำหรับ Tournament Operating System เวอร์ชันแรก โดยตั้งต้นจากการแข่งขันเกมแนวจัดอันดับ/เก็บคะแนนรวม เช่น Golden Spatula, TFT-like, Battle Royale, หรือเกมที่มีหลาย lobby และต้องรวมคะแนนหลายรอบ

## หลักคิด

- กติกาต้องเข้าใจง่ายสำหรับผู้เล่น
- แอดมินต้องจัดการได้คนเดียวหรือทีมเล็ก
- คะแนนต้องตรวจสอบย้อนหลังได้
- ลดดราม่าด้วยหลักฐาน, สถานะคะแนน, และ log การแก้ไข
- ระบบควรรองรับการแก้สูตรคะแนนในอนาคต แต่รายการแข่งหนึ่งรายการต้องล็อกสูตรก่อนเริ่มแข่ง

## รูปแบบการแข่งขันที่แนะนำสำหรับ MVP

ชื่อรูปแบบ: Group Points Qualifier

เหมาะกับ:

- รายการแข่งรายสัปดาห์
- ผู้เล่น 16-64 คน
- เกมที่จบเป็นอันดับ 1-8 ต่อ lobby
- การคัดผู้เล่นเข้ารอบด้วยคะแนนรวม

ไม่เหมาะกับ:

- เกม 1v1 ที่ควรใช้ bracket โดยตรง
- รายการใหญ่ที่ต้องมี seed ซับซ้อน
- รายการที่มีเงินรางวัลสูงและต้องมีระบบกรรมการเต็มรูปแบบ

## จำนวนผู้เล่น

ดราฟค่าเริ่มต้น:

- รับสมัครสูงสุด 64 คน
- Lobby ละ 8 คน
- ถ้าคนน้อยกว่า 64 คน ให้ระบบลดจำนวน lobby ตามจริง
- ถ้าจำนวนคนไม่หาร 8 ลงตัว ให้ใช้กติกา bye หรือ lobby ขนาดไม่เท่ากันตามที่ประกาศก่อนแข่ง

ค่า MVP ที่ง่ายที่สุดสำหรับเริ่มจริง:

- เริ่มที่ 16 หรือ 32 คนก่อน
- Lobby ละ 8 คน
- แข่ง 2-3 เกมต่อรอบ

## กติกาสมัคร

- ผู้เล่นสมัครได้ 1 ครั้งต่อรายการ
- ต้องใช้ชื่อในเกมและ UID จริง
- ห้ามใช้ UID ซ้ำในรายการเดียวกัน
- ผู้เล่นต้องเข้าช่องทางติดต่อที่กำหนด เช่น Discord หรือ Line
- การสมัครมีสถานะ:
  - รอตรวจสอบ
  - อนุมัติ
  - ไม่ผ่าน
  - สำรอง
  - ถอนตัว
- เฉพาะผู้เล่นที่อนุมัติแล้วเท่านั้นที่ถูกจัดเข้า lobby
- ถ้าผู้เล่นถอนตัวหลังปิดรับสมัคร แอดมินเลือกได้ว่าจะดึงผู้เล่นสำรองหรือให้ bye

## Check-in

แนะนำให้มี check-in แม้ใน MVP เพราะช่วยลดปัญหาคนไม่มา

ดราฟกติกา:

- เปิด check-in ก่อนแข่ง 30 นาที
- ปิด check-in ก่อนแข่ง 5 นาที
- ผู้เล่นที่ไม่ check-in ถือว่าสละสิทธิ์ เว้นแต่แอดมินอนุมัติเป็นกรณีพิเศษ
- ผู้เล่นสำรองสามารถแทนที่ผู้เล่นที่ไม่ check-in ได้ก่อนเริ่ม lobby แรก

ถ้าต้องการ MVP ง่ายที่สุด:

- ใช้การ check-in แบบ manual ใน Discord ก่อน
- ระบบบันทึกสถานะ check-in ภายหลัง

## การจัดกลุ่ม

ดราฟกติกา:

- ระบบสุ่มผู้เล่นที่อนุมัติแล้วเข้า lobby
- แอดมินสามารถปรับ lobby เองก่อนล็อกตารางแข่ง
- เมื่อเริ่มรอบแล้ว ห้ามย้ายผู้เล่นระหว่าง lobby ยกเว้นมีเหตุจำเป็นและต้องมี audit note
- รอบถัดไปจัดจากผู้เล่นที่ผ่านเข้ารอบเท่านั้น

กรณีมีข้อมูล seed:

- Seed สูงควรถูกกระจายคนละ lobby
- ถ้ายังไม่มี ranking ให้สุ่มทั้งหมดก่อน

## สูตรคะแนนที่แนะนำ

สูตรนี้เหมาะกับเกมที่วัดผลจากอันดับจบเกมเป็นหลัก

| อันดับ | คะแนน |
| :---: | ---: |
| 1 | 10 |
| 2 | 8 |
| 3 | 7 |
| 4 | 6 |
| 5 | 4 |
| 6 | 3 |
| 7 | 2 |
| 8 | 1 |

เหตุผล:

- อันดับ 1 ได้รางวัลชัดเจน
- อันดับ 2-4 ยังมีแรงจูงใจสูง
- อันดับ 5-8 ยังได้คะแนน ไม่หลุดทันที
- เหมาะกับการเล่นหลายเกมต่อรอบ

## Bonus และ Penalty

MVP แนะนำว่าอย่าเพิ่งมี bonus ซับซ้อน

ถ้าจำเป็นให้มีได้เฉพาะ:

- Bonus +1 สำหรับ achievement ที่ประกาศชัดเจนก่อนแข่ง
- Penalty -1 ถึง -5 สำหรับผิดกติกา
- Disqualification = 0 คะแนนในเกมนั้น หรือออกจากรายการ ขึ้นกับความรุนแรง

ห้ามใช้ bonus/penalty ที่ตัดสินยากหรือไม่มีหลักฐาน เพราะจะทำให้ดราม่าง่าย

## การคิดคะแนนรวม

- คะแนนรวมของรอบ = ผลรวมคะแนนทุกเกมในรอบนั้น
- ผู้เล่นที่คะแนนรวมสูงสุดใน lobby จะได้อันดับ lobby สูงสุด
- ผู้เล่นที่ผ่านเข้ารอบคิดจากอันดับคะแนนรวมใน lobby
- คะแนนรอบก่อนหน้าไม่ควรถูกนำไป Final เว้นแต่ประกาศว่าเป็นระบบสะสมข้ามรอบ

แนะนำ MVP:

- แต่ละรอบ reset คะแนนใหม่
- Final ตัดสินจากคะแนนใน Final เท่านั้น

## Tie-break

ถ้าคะแนนเท่ากัน ให้ใช้ลำดับนี้:

1. จำนวนครั้งที่ได้อันดับ 1 มากกว่า
2. อันดับเฉลี่ยดีกว่า
3. คะแนนเกมล่าสุดสูงกว่า
4. อันดับเกมล่าสุดดีกว่า
5. แข่ง tiebreak เพิ่ม 1 เกม ถ้ากระทบการเข้ารอบหรืออันดับรางวัล
6. Admin decision พร้อม audit note ถ้าไม่สามารถแข่งเพิ่มได้

## Bye และผู้เล่นไม่ครบ

แนะนำ MVP:

- ไม่ให้คะแนน bye โดยตรง
- ใช้ bye เพื่อผ่านเข้ารอบเท่านั้น ถ้าจำเป็น

เหตุผล:

- คะแนน bye อาจไม่ยุติธรรมกับผู้เล่นที่แข่งจริง
- ระบบ advance-only ลดข้อโต้แย้ง

ถ้าจำเป็นต้องให้คะแนน bye:

- ต้องประกาศก่อนเริ่มรายการ
- ใช้คะแนนเฉลี่ยขั้นต่ำของผู้เล่นที่ผ่านเข้ารอบใน lobby เดียวกัน
- ต้องมี audit note ทุกครั้ง

## การบันทึกผล

- แอดมินหรือกรรมการเป็นผู้บันทึกคะแนน
- คะแนนแรกเข้าเป็นสถานะ Pending
- คะแนนขึ้น leaderboard เป็นทางการหลัง Approved
- ต้องบันทึกข้อมูลขั้นต่ำ:
  - Tournament
  - Round
  - Lobby
  - Game number
  - Player
  - Placement
  - Points
  - Evidence reference

## หลักฐานผลการแข่งขัน

หลักฐานที่รับได้:

- Screenshot หน้าสรุปผลเกม
- วิดีโอ replay หรือคลิปช่วงจบเกม
- ข้อมูลจากกรรมการใน lobby
- OCR จากภาพ ใช้ได้เฉพาะเป็นตัวช่วย ไม่ใช่ผลทางการทันที

กติกา:

- ถ้าคะแนนไม่ตรงกับหลักฐาน ให้ใช้หลักฐานที่ชัดเจนกว่า
- ถ้าหลักฐานขัดแย้งกัน ให้กรรมการหรือ Tournament Admin ตัดสิน
- ทุกการแก้คะแนนต้องมี note

## การโต้แย้งคะแนน

ดราฟกติกา:

- ผู้เล่นยื่นโต้แย้งได้ภายใน 15 นาทีหลังประกาศคะแนนรอบนั้น
- ต้องระบุเกม, lobby, คะแนนที่ผิด, และหลักฐาน
- คะแนนที่ถูกโต้แย้งจะมีสถานะ Disputed
- Admin หรือ Referee ต้องตัดสินก่อน finalize รอบ
- ผลการตัดสินต้องมีเหตุผลสั้น ๆ

## การประกาศผล

- Leaderboard public แสดงเฉพาะข้อมูลที่ไม่ใช่ข้อมูลส่วนตัว
- คะแนน Pending ต้องแสดงชัดว่าไม่ใช่คะแนน final
- คะแนน Approved ใช้คำนวณการเข้ารอบ
- คะแนน Final ห้ามแก้ไข ยกเว้น Super Admin พร้อม audit note

## การตัดสิทธิ์

ตัวอย่างเหตุที่อาจถูกตัดสิทธิ์:

- ใช้บัญชีแทนกัน
- ใช้โปรแกรมช่วยเล่นหรือโกง
- ไม่มารายงานตัว/check-in
- จงใจส่งข้อมูลเท็จ
- รบกวนการแข่งขันหรือผู้เล่นอื่น
- ไม่ปฏิบัติตามคำตัดสินของกรรมการ

บทลงโทษ:

- Warning
- Penalty points
- 0 คะแนนในเกมนั้น
- ตัดสิทธิ์จากรอบนั้น
- ตัดสิทธิ์จากรายการ

## สิ่งที่ต้องยืนยันก่อนใช้งานจริง

- เกมแรกที่ใช้จัดแข่ง
- จำนวนผู้เล่นเป้าหมาย
- จำนวนเกมต่อรอบ
- สูตรคะแนนสุดท้าย
- มี check-in หรือไม่
- ใช้คะแนนสะสมข้ามรอบหรือ reset คะแนนทุกรอบ
- วิธีจัดการผู้เล่นหลุด/เกมหลุด
- เวลาสำหรับยื่น dispute
- ใครมีสิทธิ์ approve/finalize คะแนน

## Recommendation

สำหรับเริ่มรายการแรก แนะนำใช้:

- 16 หรือ 32 คน
- Solo
- Lobby ละ 8 คน
- 2 เกมต่อรอบคัดเลือก
- 3 เกมรอบ Final
- คะแนนตามอันดับ 10/8/7/6/4/3/2/1
- Reset คะแนนทุกรอบ
- Top 4 ผ่านเข้ารอบ
- Check-in manual ก่อน
- Score approval manual
- Dispute ภายใน 15 นาที

ชุดนี้เล็กพอให้จัดจริงได้ แต่ครบพอสำหรับออกแบบ database, API, leaderboard, audit log, และระบบตรวจสอบคะแนนต่อได้
