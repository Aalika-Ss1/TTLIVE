# TTLIVE Roadmap

Roadmap นี้ออกแบบสำหรับการทำคนเดียวก่อน แล้วค่อยขยายเป็นทีมในอนาคต

## Phase 0: Foundation

สถานะ: กำลังทำ

เป้าหมาย:

- จัดโครง repo
- กำหนดมาตรฐานไฟล์
- แยก asset, template, data, output
- วางระบบตารางไลฟ์

ผลลัพธ์ที่ต้องมี:

- `README.md`
- `DASHBOARD.md`
- `00_company/system-master-plan.md`
- `03_live_operations/stream-schedule-system.md`
- template ตารางไลฟ์
- data เกมหลัก

## Phase 1: Consistent Live

เป้าหมาย:

- ไลฟ์ให้สม่ำเสมอก่อน
- ทุกครั้งต้องมี checklist และ report สั้น
- ไลฟ์ขั้นต่ำ 1 ชั่วโมง

งานที่ต้องทำ:

- สร้าง `before-live` และ `after-live` checklist ให้ใช้จริง
- กำหนดตารางประจำสัปดาห์
- สร้าง live report template
- สร้าง experiment log สำหรับเวลาไลฟ์

ตัวชี้วัด:

- จำนวนครั้งที่ไลฟ์ต่อสัปดาห์
- จำนวนไลฟ์ที่เกิน 1 ชั่วโมง
- จำนวน report ที่กรอกครบ

## Phase 2: Clip Pipeline

เป้าหมาย:

- ทุก live ควรมี clip candidate อย่างน้อย 1 ช่วง
- สร้าง workflow ตัดคลิปที่ไม่หนักเกินไป

งานที่ต้องทำ:

- สร้าง clip candidate log
- สร้าง SOP ตัดคลิปสั้น
- สร้าง SOP อัปโหลดคลิป
- สร้าง caption template สำหรับคลิป

ตัวชี้วัด:

- จำนวน clip candidate ต่อ live
- จำนวนคลิปที่โพสต์ต่อสัปดาห์
- ยอดดู/retention/comment ของคลิป

## Phase 3: Visual System

เป้าหมาย:

- มีภาพประกาศก่อนขึ้นไลฟ์
- มีภาพหลังลงไลฟ์หรือ recap
- มี overlay/layout ระหว่างไลฟ์
- ปกและตารางดูเป็นแบรนด์เดียวกัน

งานที่ต้องทำ:

- กำหนด live starting soon layout
- กำหนด live offline/recap layout
- กำหนด in-live overlay layout
- เก็บ asset จริง เช่น logo, icon เกม, mascot

ตัวชี้วัด:

- จำนวน asset ที่ใช้ซ้ำได้
- เวลาในการทำภาพต่อครั้งลดลง
- ภาพทุกชิ้นใช้ format เดียวกัน

## Phase 4: Analytics And Improvement

เป้าหมาย:

- รู้ว่าเกมไหน เวลาไหน และ format ไหนเวิร์ก
- ตัดสินใจด้วยข้อมูล ไม่ใช่ความรู้สึกอย่างเดียว

งานที่ต้องทำ:

- live analytics template
- clip performance template
- monthly review
- experiment tracking

ตัวชี้วัด:

- average viewers
- peak viewers
- follower gain
- chat activity
- clips posted
- best performing clip

## Phase 5: Automation

เป้าหมาย:

- ให้ Codex/script ช่วยงานซ้ำ
- คนทำงานกรอกข้อมูลน้อยลง

งานที่ต้องทำ:

- generate schedule image
- generate live cover
- generate captions
- summarize report
- prepare weekly plan from data

ข้อควรระวัง:

- automation ต้องตามมาตรฐาน ไม่ใช่สร้าง style ใหม่ทุกครั้ง
- ถ้า automation ทำให้แก้งานยากกว่าเดิม ให้กลับไปใช้ template manual ก่อน

## Phase 6: Team And Monetization

เป้าหมาย:

- เตรียมรับคนช่วย
- เตรียม sponsor หรือรายได้อื่น

งานที่ต้องทำ:

- role assignment
- moderator workflow
- sponsor package
- asset license tracking
- access inventory
